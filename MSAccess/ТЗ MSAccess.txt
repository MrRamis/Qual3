4. Требования к автоматизированной системе

4.1. Требования к структуре АС в целом

4.1.1. Перечень подсистем (при их наличии), их назначение и основные характеристики.
MS Access дает возможность создания различных объектов баз данных, таких как таблицы, запросы, формы, отчеты и макросы.

4.1.2. Требования к способам и средствам обеспечения информационного взаимодействия компонентов АС
MS Access должен позволять создавать связи между объектами баз данных и устанавливать правила для их взаимодействия.

4.1.3. Требования к характеристикам взаимосвязей создаваемой АС со смежными АС, требования к интероперабельности, требования к ее совместимости, в том числе указания о способах обмена информацией
MS Access должен взаимодействовать с другими базами данных и приложениями Microsoft, обеспечивая интеграцию данных и информационный обмен.

4.1.4. Требования к режимам функционирования АС

Оптимальная производительность:
Использование индексов и оптимизация SQL-запросов для быстрого доступа к данным.
Регулярная очистка и компактация базы данных для поддержания ее эффективной работы.

Надежность и безопасность:
Создание бэкапов базы данных для предотвращения потери данных в случае сбоев.
Установка паролей на базу данных и разграничение прав доступа к различным пользователям.

Масштабируемость:
Разделение базы данных на отдельные таблицы и связывание их для удобного масштабирования данных.
Использование связей между таблицами для обеспечения целостности данных и удобства работы с расширяющимися объемами информации.

Пользовательский интерфейс:
Создание удобного и интуитивно понятного пользовательского интерфейса с использованием форм и отчетов.
Добавление элементов управления для удобства работы пользователей и оптимизации процесса ввода и анализа данных.

4.1.5. Требования по диагностированию АС

Мониторинг производительности:
Регулярный мониторинг работы базы данных MS Access для выявления узких мест и оптимизации запросов.
Отслеживание нагрузки на сервер и процессор для обеспечения стабильной производительности.

Проверка целостности данных:
Выполнение регулярных проверок целостности данных для выявления возможных ошибок и проблем в базе данных.
Анализ и исправление дублирующихся записей, ошибок целостности и отсутствующих значений.

Мониторинг безопасности:
Анализ журналов доступа к базе данных для обнаружения несанкционированных попыток доступа.
Проведение аудита безопасности для проверки соответствия прав доступа согласно установленным требованиям.

Обеспечение резервного копирования:
Ежедневное создание бэкапов базы данных MS Access для предотвращения потери информации.
Проверка и восстановление данных из резервных копий в случае необходимости.

4.1.6. Перспективы развития, модернизации АС

Переход на более мощные СУБД:
Рассмотрение возможности перехода с MS Access на более масштабируемые и производительные системы управления базами данных, такие как MySQL, PostgreSQL или MS SQL Server.
Проведение анализа стоимости и потенциальных выгод от такого перехода.

Внедрение облачных технологий:
Исследование и внедрение облачных решений для хранения и анализа данных, что позволит компании избежать проблем с локальным хранилищем и получить высокую масштабируемость.
Рассмотрение интеграции с облачными сервисами, такими как AWS, Google Cloud или Microsoft Azure.

Оптимизация пользовательского интерфейса:
Разработка нового пользовательского интерфейса системы, обеспечивающего более интуитивное и удобное взаимодействие пользователя с данными.
Использование современных подходов к дизайну пользовательских интерфейсов для улучшения опыта пользователей.

Улучшение безопасности данных:
Внедрение дополнительных мер безопасности, таких как двухфакторная аутентификация, шифрование данных и мониторинг утечек информации.
Регулярное обновление механизмов защиты и адаптация к новым угрозам информационной безопасности.

4.3. Требования к видам обеспечения АС

4.3.1. Требования к математическому обеспечению АС

Итерационные методы и численное моделирование для решения задач оптимизации

4.3.2. Требования к информационному обеспечению

Возможность интеграции с другими информационными системами для обмена данными.
Совместимость с различными устройствами и платформами для удобства использования.
Возможность быстрой настройки и добавления новых функциональных возможностей.

4.3.3. Требования к лингвистическому обеспечению

Не требуется

4.3.4. Требования к программному обеспечению

Язык программирования: Visual Basic for Applications (VBA).
Встроенные библиотеки для создания форм и отчетов, а также реализации математических операций
Требования к ОС: MS Access работает на операционных системах Windows, требования к ОС варьируются в зависимости от версии программы.

4.3.5. Требования к техническому обеспечению

Минимальные требования:

ОЗУ: Рекомендуется иметь не менее 4 ГБ оперативной памяти.
Жесткий диск: Рекомендуется использование 128 ГБ SSD для ускорения работы.
Периферийные устройства: Монитор, клавиатура, мышь - стандартные устройства для работы с MS Access.

4.3.6. Требования к метрологическому обеспечению

Безопасность данных: Гарантия сохранности и целостности информации путем регулярного резервного копирования и установки парольной защиты.
Метаданные: Корректное определение полей, типов данных и связей между таблицами.

4.3.7. Требования к организационному обеспечению

4.4. Общие технические требования к АС

4.4.1. Требования к численности и квалификации персонала и
пользователей АС

Численность персонала включает в себя администратора базы данных, разработчика приложения и техническую поддержку

Квалификация персонала:
Администратор базы данных должен иметь глубокие знания MS Access, SQL и баз данных в целом.
Разработчик приложения должен обладать навыками проектирования баз данных, создания запросов и отчетов, а также знаниями VBA для автоматизации.
Техническая поддержка должна быть готова решать оперативные проблемы пользователей и обеспечивать обучение и консультации.

Квалификация пользователей:
Пользователи должны иметь базовое понимание работы с базами данных и умения вводить и извлекать данные.

4.4.2. Требования к показателям назначения
Время выполнения запроса: не более 2 секунд на простой запрос

4.4.3. Требования к надежности

Резервное копирование данных: Регулярное создание резервных копий базы данных и ее компонентов
Защита данных: Гарантированное соблюдение принципов безопасности и конфиденциальности данных
Восстановление данных

4.4.4. Требования по безопасности

Установка аутентификации и авторизации:
Организация процессов аутентификации для идентификации пользователей и авторизации для установления уровней доступа к данным и функциональности

Шифрование данных:
Использование шифрования для защиты данных, передаваемых по сети или хранимых в базе данных, чтобы предотвратить их перехват и несанкционированное использование.

Управление доступом:
Настройка точечного управления доступом к данным и приложениям, чтобы ограничить права пользователей согласно их роли и обязанностям.

4.4.5. Требования к эргономике и технической эстетике

Разработка удобного и понятного интерфейса для пользователей системы
Обеспечение легкой и интуитивной навигации по системе, с ясными пунктами меню
Использование одной цветовой схемы и стиля для всех элементов формы, чтобы создать единый визуальный образ.
Выбор читаемых шрифтов и оптимального размера текста для легкости восприятия.

4.4.6. Требования к транспортабельности для подвижных АС

АС не является подвижной

4.4.7. Требования к эксплуатации, техническому обслуживанию, ремонту и хранению компонентов АС

Регулярно обучать новых пользователей вводить, редактировать и удалять данные в базе данных MS Access, чтобы предотвратить ошибочные действия и повысить производительность.
Раз в месяц проводить аудит базы данных MS Access на предмет устаревших объектов, неиспользуемых запросов и отчетов, чтобы улучшить ее эффективность.
Настроить автоматическое создание резервных копий базы данных MS Access каждый вечер и перед важными обновлениями, чтобы минимизировать потерю данных.
Хранить копии баз данных MS Access на отдельных носителях данных

4.4.8. Требования к защите информации от несанкционированного доступа

Создание различных уровней доступа для пользователей в базе данных MS Access: администратор с полными правами, операторы с правом на чтение и запись, гости с ограниченным доступом.

4.4.9. Требования по сохранности информации при авариях

Регулярное резервное копирование
План восстановления данных: Разработка документированного плана восстановления данных для базы данных MS Access

4.4.10. Требования к защите от влияния внешних воздействий

Установка антивирусного программного обеспечения на сервер с базой данных MS Access для регулярного сканирования данных на наличие вирусов и вредоносного кода
Регулярное обновление Microsoft Access и других системных компонентов для закрытия уязвимостей безопасности

4.4.11. Требования к патентной чистоте и патентоспособности

Не требуется

4.4.12. Требования по стандартизации и унификации

Соблюдение общепринятых стандартов разработки программного обеспечения
Cоблюдение однородности формата и стиля кода при написании макросов, запросов и модулей VBA
Применение принципов нормализации данных при проектировании базы данных MS Access

4.4.13. Дополнительные требования
